## Simple Thread Pool

**Thread pool** нужен, потому что создание отдельного `Thread` под каждую мелкую задачу обходится дорого. В этой учебной
реализации заранее создаётся несколько **рабочих потоков**, а задачи добавляются в общую очередь
`Queue<Action>`.

Так как `Queue<Action>` не является **потокобезопасной**, доступ к ней защищён через `_lock`. Если задач нет, рабочий
поток не крутится в бесконечном цикле и не нагружает CPU, а засыпает через `Monitor.Wait`.
Когда новая задача добавляется в очередь, вызывается `Monitor.Pulse`, и один из спящих потоков просыпается, забирает
задачу и выполняет её.

Для корректного завершения используется `Dispose`, устанавливается флаг `_isStopping`, затем `Monitor.PulseAll` будит
все спящие потоки, чтобы они увидели этот флаг и вышли из рабочего цикла.
После этого `Join` просто дожидается, пока каждый поток сам завершит выполнение.Такой подход лучше, чем `Abort`, потому
что поток не прерывается насильно в случайном месте, а спокойно доходит до
безопасной точки завершения.
